# long과 Long의 차이

`long`과 `Long`은 둘 다 자바에서 64비트 정수를 표현하지만, 몇 가지 중요한 차이점이 있습니다.

## long

- **기본형 (Primitive Type)**: `long`은 자바의 기본 자료형 중 하나입니다.
- **메모리 사용**: 64비트(8바이트)를 사용합니다.
- **기본 값**: `long` 변수의 기본 값은 `0`입니다.
- **성능**: 기본형이기 때문에 객체 포장 비용이 없어서 메모리 사용과 성능 면에서 더 효율적입니다.
- **연산 지원**: 기본형이므로 산술 연산, 비교 연산 등을 바로 수행할 수 있습니다.

### 사용 예:
```java
long a = 123456789L;
long b = 987654321L;
long c = a + b;
```

## Long
- **참조형 (Wrapper Class)**: `Long`은 java.lang 패키지에 속한 래퍼 클래스입니다. 기본형 `long`을 객체로 감쌉니다.
- **메모리 사용**: 객체이기 때문에 추가적인 메모리 오버헤드가 있습니다. `Long` 객체는 기본적으로 16바이트의 메모리를 사용하며, 이는 8바이트의 값과 추가적인 객체 헤더 오버헤드를 포함합니다.
- **기본 값**: `Long` 객체는 `null`로 초기화될 수 있습니다.
- **성능**: 객체 포장 비용이 있어서 기본형보다 성능이 떨어질 수 있습니다.
- **유용성**: `Long` 클래스는 여러 유용한 메서드를 제공합니다. 예를 들어, `Long.parseLong(String s)`, `Long.valueOf(long l)` 등.

### 사용 예:
```java
Long a = 123456789L;
Long b = 987654321L;
Long c = a + b; // 자동 언박싱 및 박싱 발생
```

## 비교

| 특성         | long       | Long                        |
|--------------|------------|-----------------------------|
| **타입**     | 기본형     | 참조형                      |
| **메모리 사용** | 8바이트    | 16바이트 이상                |
| **초기 값**   | 0          | null                        |
| **성능**     | 높음       | 낮음                        |
| **메서드 제공** | 없음       | 다양한 유틸리티 메서드         |
| **사용 예**  | `long a = 1;` | `Long a = 1L;`           |

## 사용 시기
- **long**: 성능이 중요한 경우, 기본형을 사용하는 것이 좋습니다. 숫자 연산이 많고 객체로 포장하는 것이 필요하지 않을 때 사용합니다.
- **Long**: 객체로서 사용해야 할 때, 예를 들어 컬렉션과 함께 사용할 때, 혹은 `null` 값이 필요할 때 사용합니다. 또한, `Long` 클래스의 유틸리티 메서드가 필요할 때 사용합니다.

### 예제 코드 비교
#### long 사용 예

```java
public class Main {
    public static void main(String[] args) {
        long a = 123456789L;
        long b = 987654321L;
        long c = a + b;
        System.out.println("Sum: " + c);
    }
}
```

#### Long 사용 예

```java
public class Main {
    public static void main(String[] args) {
        Long a = 123456789L;
        Long b = 987654321L;
        Long c = a + b; // 자동 언박싱 및 박싱 발생
        System.out.println("Sum: " + c);
    }
}
```
